# Incident response

1. Detect and record a security event with UTC time and correlation identifiers.
2. Triage severity and activate the approved response team.
3. Contain by revoking sessions/credentials, disabling integrations, or restricting network paths without destroying evidence.
4. Preserve sanitized logs, deployment metadata, relevant audit history, and database evidence under legal guidance.
5. Eradicate, rotate secrets/certificates, remediate, and validate tenant isolation.
6. Restore from verified sources and monitor closely.
7. Complete contractual/regulatory notification analysis and a blameless post-incident review.

Never place patient or prohibited information in ordinary tickets or application incident records. Emergency contacts, escalation times, notification duties, counsel, forensics provider, insurer, and communications approvals must be completed by each production operator.
